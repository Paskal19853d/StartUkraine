Анализ проблемы и инструкция по исправлению
После детального изучения кода репозитория я нашёл точную причину смещения точек. Объясняю по шагам.

🔍 В чём проблема
Две разные системы координат:

admin.html (редактор карты) — использует SVG:

SVG карта имеет viewBox="0 0 14000 9500"
Функция screen2svg() переводит клик в SVG-координаты с учётом реальных размеров SVG-контейнера
При сохранении: pos_x = svgX / 14000, pos_y = svgY / 9500 — нормализованные координаты относительно SVG viewBox
index.html (пользовательский вид) — использует Canvas:

Функция w2s(xr, yr) = {x: xr * W * tr.s + tr.x, y: yr * H * tr.s + tr.y}
W и H — ширина и высота всего Canvas (он занимает весь экран)
Точка рисуется как p.x * W, то есть нормализованное значение умножается на ширину экрана
Проблема: SVG карта в index.html имеет preserveAspectRatio="xMidYMid meet". Это значит, что при соотношении сторон экрана, отличном от 14000:9500 (≈ 1.47:1), карта масштабируется с сохранением пропорций и не заполняет Canvas целиком — появляются пустые полосы сверху/снизу или слева/справа (letterboxing). Но Canvas рисует точки по всей своей ширине/высоте без учёта этих полос.

📁 Файл для исправления
index.html — единственный файл который нужно изменить.

🛠 Пошаговая инструкция
Шаг 1. Найди функции трансформации (строки ~2313–2314)
Сейчас код выглядит так:

function s2w(sx,sy) { return {x:(sx-tr.x)/(tr.s*W), y:(sy-tr.y)/(tr.s*H)}; }

Шаг 3. Добавь функцию получения реального положения SVG карты
После строки с function resize() добавь вспомогательную функцию getSvgBounds():

js
function getSvgBounds() {
  // SVG viewBox: 14000 x 9500, preserveAspectRatio = xMidYMid meet
  const SVG_W = 14000, SVG_H = 9500;
  const svgAspect = SVG_W / SVG_H;  // ≈ 1.4737
  const canvasAspect = W / H;

  let svgDrawW, svgDrawH, svgOffX, svgOffY;

  if (canvasAspect > svgAspect) {
    // Canvas шире — пустые полосы слева и справа
    svgDrawH = H;
    svgDrawW = H * svgAspect;
    svgOffX = (W - svgDrawW) / 2;
    svgOffY = 0;
  } else {
    // Canvas выше — пустые полосы сверху и снизу
    svgDrawW = W;
    svgDrawH = W / svgAspect;
    svgOffX = 0;
    svgOffY = (H - svgDrawH) / 2;
  }

  return { x: svgOffX, y: svgOffY, w: svgDrawW, h: svgDrawH };
}
Шаг 4. Исправь функции w2s и s2w
Замени существующие функции на:

js
function w2s(xr, yr) {
  const b = getSvgBounds();
  return {
    x: (b.x + xr * b.w) * tr.s + tr.x,
    y: (b.y + yr * b.h) * tr.s + tr.y
  };
}

function s2w(sx, sy) {
  const b = getSvgBounds();
  return {
    x: ((sx - tr.x) / tr.s - b.x) / b.w,
    y: ((sy - tr.y) / tr.s - b.y) / b.h
  };
}
Шаг 5. Проверь applyTr (строка ~2305)
Убедись что applyTr применяет transform к sl (SVG-слой):

js
function applyTr() { 
  sl.style.transform = `translate(${tr.x}px,${tr.y}px) scale(${tr.s})`; 
  sl.style.transformOrigin = '0 0';
}
Это должно быть уже правильно.

✅ Альтернативное простое решение (если хочешь минимальный патч)
Если не хочешь переписывать w2s/s2w, можно сделать так: в CSS задать SVG карте явное соотношение сторон через контейнер чтобы Canvas и SVG имели одинаковые пропорции. Найди в Style.css или в <style> блоке стили для #map-wrap, #svg-layer, #canvas-layer и сделай так:

css
#map-wrap {
  aspect-ratio: 14000 / 9500; /* фиксируем пропорцию карты */
  margin: auto;
}
Но этот подход изменит layout страницы, поэтому первый вариант (правка w2s/s2w) надёжнее.
